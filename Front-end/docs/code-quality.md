# Code Quality

## Principles

- SOLID
- Clean Architecture
- Clean Code
- DRY
- KISS
- Separation of Concerns
- Feature Isolation
- Dependency Inversion

## Type safety

- use strict TypeScript
- avoid `any`
- define explicit types for requests, responses, and shared contracts

## Reuse

- prefer reusable components over duplicated UI
- prefer reusable hooks over repeated state logic
- prefer reusable services over hardcoded feature logic

## Maintenance rules

- avoid duplicated code
- keep logic close to the correct layer
- keep components focused on presentation
- keep business rules out of the view layer
