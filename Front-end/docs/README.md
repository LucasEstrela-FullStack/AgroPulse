# AgroPulse Plans

This folder now groups the front-end plan into focused documents, one topic per file.

## Reading order

1. `overview.md` - product vision, scope, pages, and deliverables
2. `architecture.md` - stack, architecture, scale, and data layer
3. `ui-ux.md` - design language and visual direction
4. `code-quality.md` - engineering standards and implementation rules
5. `devops.md` - automation, CI/CD, and release requirements
6. `testing.md` - test strategy and quality targets

## Folder map

- `overview.md`: what AgroPulse is and what the first release should include
- `architecture.md`: how the front-end should be structured
- `ui-ux.md`: how the interface should look and behave
- `code-quality.md`: how the codebase should stay maintainable
- `devops.md`: how the project should be built and delivered
- `testing.md`: how quality should be validated
